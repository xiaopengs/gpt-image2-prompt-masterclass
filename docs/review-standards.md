# GPT Image 2 提示词教程 — 内容审核标准与流程

> 版本：v1.0 | 创建日期：2026-04-26 | 用途：教程内容质量审核

---

## 一、审核目标

确保《GPT Image 2 提示词教程》在内容准确性、教学设计、语言表达、案例质量、美学描述、排版规范和知识覆盖七个维度上达到出版级质量标准，适合零基础用户学习使用。

---

## 二、审核维度与评分标准

### 总分：35分（7项 × 5分）

| 分值 | 等级 | 含义 |
|------|------|------|
| 5分 | 优秀 | 完全达标，无需修改 |
| 4分 | 良好 | 基本达标，有微小瑕疵可快速修正 |
| 3分 | 合格 | 达到最低标准，需要修改但不影响整体框架 |
| 2分 | 不合格 | 存在明显问题，需要较大修改 |
| 1分 | 严重缺陷 | 存在根本性错误或缺失，需要重写 |

---

### 2.1 内容准确性（权重：高）

**审核要点：**

1. **技术描述正确性**
   - GPT Image 2 的功能边界描述是否准确（能做什么、不能做什么）
   - 参数说明是否与官方文档一致（如尺寸、质量模式等）
   - 提示词语法规则描述是否正确（如自然语言描述方式、风格关键词等）

2. **版本一致性**
   - 是否标注了适用的产品版本
   - 截图/界面是否与当前版本一致
   - 是否注明了可能随版本变化的功能

3. **事实核查**
   - 引用的外部数据/案例是否真实可验证
   - 与竞品的对比描述是否客观公正
   - 对 AI 生成图像的局限性描述是否诚实

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 无任何技术错误，所有描述经过验证 |
| 4 | 存在 1-2 处措辞不够精确，但不影响理解 |
| 3 | 有少量技术细节需更新或补充 |
| 2 | 存在明显技术错误（如错误的参数值、过时的功能描述） |
| 1 | 核心技术概念描述错误，会误导读者 |

---

### 2.2 教学设计（权重：高）

**审核要点：**

1. **学习路径合理性**
   - 是否遵循"由浅入深、循序渐进"原则
   - 章节之间是否有逻辑递进关系
   - 前置知识是否在需要之前已讲解

2. **教学目标明确性**
   - 每章/每节是否有明确的学习目标（学完能做什么）
   - 学习目标是否可衡量、可验证

3. **练习与实操**
   - 每个知识点后是否有配套练习
   - 练习难度是否与当前章节匹配
   - 是否提供练习参考答案或示例输出
   - 是否设计了"模仿→修改→创造"的三阶段练习

4. **认知负荷控制**
   - 单次引入的新概念不超过 3-5 个
   - 复杂概念是否有类比或分步拆解
   - 是否使用"先看效果，再讲原理"的教学策略

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 学习路径清晰，练习充分，目标明确，认知负荷合理 |
| 4 | 整体路径合理，练习可再丰富一些 |
| 3 | 有练习但不够，或章节顺序有小问题 |
| 2 | 缺乏练习，或难度跨度太大 |
| 1 | 无教学设计可言，知识堆砌无逻辑 |

---

### 2.3 语言质量（权重：高）

**审核要点：**

1. **通俗易懂性**
   - 小白（无 AI 绘画经验）能否看懂？
   - 专业术语首次出现时是否有简明解释
   - 是否避免了不必要的行话和英文缩写

2. **表述一致性**
   - 同一概念全文是否使用统一术语
   - 人称是否一致（建议用"你"而非"您"或"我们"）
   - 语气是否亲切友好，不居高临下

3. **简洁性**
   - 是否存在冗余重复的段落
   - 每句话是否都有信息增量
   - 能用一句话说清的不要用一段话

4. **可读性**
   - 段落长度是否适中（建议不超过 5-6 行）
   - 是否善用列表、表格替代长段落
   - 关键信息是否突出（加粗、提示框等）

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 小白轻松看懂，阅读体验流畅，无歧义 |
| 4 | 个别地方稍显晦涩，稍加调整即可 |
| 3 | 整体能看懂，但部分段落需要反复读 |
| 2 | 术语过多，小白阅读困难 |
| 1 | 语言晦涩，读者会大量跳读或放弃 |

---

### 2.4 案例质量（权重：高）

**审核要点：**

1. **有效性**
   - 提示词案例是否能稳定生成预期效果
   - 是否标注了生成条件（模型版本、参数设置）
   - 是否提供了失败案例及其改进方法

2. **可复现性**
   - 读者复制提示词能否得到相似结果
   - 是否说明了结果可能存在的随机性
   - 是否提供了完整的提示词（而非截断或模糊描述）

3. **多样性**
   - 案例是否覆盖不同风格/场景/技巧
   - 是否有从简单到复杂的梯度案例
   - 是否包含常见的反面教材（what not to do）

4. **教学价值**
   - 每个案例是否明确展示了某个知识点
   - 案例是否有逐步拆解分析（为什么这样写）
   - 是否鼓励读者在案例基础上修改实验

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 案例丰富、可复现、有正反对比，教学价值极高 |
| 4 | 案例充足，大部分可复现 |
| 3 | 有案例但覆盖面不够，或缺少分析 |
| 2 | 案例太少，或案例与讲解脱节 |
| 1 | 缺乏有效案例，或案例不可复现 |

---

### 2.5 美学描述（权重：中）

**审核要点：**

1. **艺术/设计术语准确性**
   - 光影描述术语是否准确（如：rim light、golden hour、chiaroscuro）
   - 构图术语是否正确（如：rule of thirds、leading lines、negative space）
   - 风格描述是否精准（如：photorealistic、watercolor、low-poly）

2. **色彩描述专业性**
   - 色彩术语使用是否准确（如：complementary colors、warm tones）
   - 是否提供了足够的视觉引导（让读者能想象出画面）

3. **提示词美学模板**
   - 是否提供了结构化的提示词模板（主体 + 环境 + 光影 + 风格 + 情绪）
   - 模板是否易于理解和套用

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 术语精准，美学描述体系完整，模板实用 |
| 4 | 术语基本准确，偶有不够精确的描述 |
| 3 | 有美学相关内容，但深度不够 |
| 2 | 美学描述混乱，术语使用有误 |
| 1 | 完全缺乏美学维度的指导 |

---

### 2.6 排版规范（权重：中）

**审核要点：**

1. **结构层次**
   - 标题层级是否规范（H1 → H2 → H3，不跳级）
   - 章节长度是否均衡（避免某章过长某章过短）
   - 是否有清晰的导航/目录

2. **视觉排版**
   - 提示词是否用代码块或特殊样式突出显示
   - 图片是否有编号和说明文字
   - 重要提示/警告是否有醒目的标注
   - 表格是否对齐、易读

3. **格式一致性**
   - 全文的标记符号使用是否统一
   - 图片尺寸/样式是否统一
   - 中英文混排是否有空格（如：`使用 GPT Image 2 生成`）

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 结构清晰，排版美观，格式完全统一 |
| 4 | 整体规范，有 1-2 处格式不统一 |
| 3 | 基本可读，排版有改进空间 |
| 2 | 排版混乱，阅读体验差 |
| 1 | 无结构可言，纯文本堆砌 |

---

### 2.7 完整性（权重：高）

**审核要点：**

1. **大纲覆盖率**
   - 对照教程大纲，检查每个知识点是否都有对应内容
   - 是否有遗漏的重要主题

2. **学习闭环**
   - 是否有开篇导读（告诉读者学完能得到什么）
   - 是否有章节小结（巩固重点）
   - 是否有综合实战项目（串联所有知识点）
   - 是否有常见问题/排错指南

3. **配套资源**
   - 是否提供了提示词速查表/cheat sheet
   - 是否有推荐进阶资源
   - 是否有社区/讨论渠道引导

**评分细则：**

| 分值 | 标准 |
|------|------|
| 5 | 大纲100%覆盖，学习闭环完整，配套资源丰富 |
| 4 | 大纲基本覆盖，缺少少量非核心内容 |
| 3 | 覆盖约80%，有部分重要内容缺失 |
| 2 | 覆盖不足60%，明显不完整 |
| 1 | 严重缺失，大量大纲内容未涉及 |

## 四、审核 SOP（标准操作流程）

### 阶段一：准备（约 30 分钟）

1. **获取物料**
   - 获取教程完整文档（或文档链接）
   - 获取教程大纲/目录结构
   - 获取 GPT Image 2 官方文档作为参考基准

2. **准备工具**
   - 打开本文档（审核标准）
   - 准备评分表（可用上表的副本）
   - 准备批注工具（文档批注/评论功能）

3. **设定审核范围**
   - 确认本次审核是"初审"还是"复审"
   - 初审：全面审核所有七个维度
   - 复审：重点审核上一轮标记的修改项

### 阶段二：通读（约 60 分钟）

4. **第一遍：快速通读**
   - 不做批注，纯阅读
   - 记录整体印象：流程是否顺畅？哪里卡壳？
   - 标记"明显好"和"明显差"的地方

5. **第二遍：对照大纲**
   - 逐项对照大纲，标记遗漏的知识点
   - 检查章节之间的逻辑关系
   - 记录完整性问题

### 阶段三：逐项审核（约 90 分钟）

6. **内容准确性审核**
   - 逐一验证技术描述（可对照官方文档）
   - 测试关键提示词案例是否能得到预期结果
   - 记录所有事实性错误

7. **教学设计审核**
   - 绘制知识点依赖图，检查是否有"未学先用的概念"
   - 评估练习的充分性和梯度
   - 检查学习目标是否达成

8. **语言质量审核**
   - 抽读给非技术背景的人（或模拟小白视角）
   - 标记晦涩难懂的段落
   - 检查术语一致性

9. **案例质量审核**
   - 实际复制提示词到 GPT Image 2 测试
   - 记录生成结果与描述的匹配度
   - 评估案例的多样性和教学价值

10. **美学描述审核**
    - 检查专业术语准确性
    - 评估提示词模板的实用性

11. **排版规范审核**
    - 检查标题层级
    - 检查格式一致性
    - 检查视觉元素

12. **完整性审核**
    - 对照大纲逐项打勾
    - 检查学习闭环要素

### 阶段四：评分与报告（约 30 分钟）

13. **填写评分表**
    - 每个维度给出具体分数
    - 每个分数必须有具体的依据（引用原文）

14. **撰写审核报告**
    - 使用以下模板：

```markdown
# 审核报告

**审核人：** ____
**审核日期：** ____
**审核版本：** ____
**总分：** __/35（等级：__）

## 优点（做得好的地方）
1. ...
2. ...

## 问题清单（按严重程度排序）

### 🔴 严重问题（必须修改）
- [ ] 问题描述 | 位置：章节X | 建议：...

### 🟡 一般问题（建议修改）
- [ ] 问题描述 | 位置：章节X | 建议：...

### 🟢 改进建议（可选优化）
- [ ] 建议描述 | 位置：章节X

## 各维度评分明细
（填入评分表）

## 总结与建议
...
```

15. **交付审核报告**
    - 将报告发送给教程作者
    - 安排复审时间（通常修改后 2-3 个工作日）

### 阶段五：复审（约 45 分钟）

16. **检查修改项**
    - 逐项确认每个问题是否已解决
    - 对新增内容进行审核

17. **更新评分**
    - 重新打分
    - 对比前后改善幅度

18. **最终确认**
    - 总分达到 B 级（25分）以上 → 通过审核，可发布
    - 未达标 → 返回修改，安排下一轮复审

## 五、常见问题速查

| 问题类型 | 典型表现 | 建议处理方式 |
|----------|---------|-------------|
| 技术过时 | 引用了已废弃的功能 | 标注 🔴，需立即更新 |
| 提示词失效 | 案例无法生成预期效果 | 标注 🔴，需重新测试并替换 |
| 术语混乱 | 同一概念用了3种不同叫法 | 标注 🟡，统一术语表 |
| 难度跳跃 | 前一节讲基础，下一节突然高级 | 标注 🔴，增加过渡内容 |
| 缺乏练习 | 连续3页纯理论无实操 | 标注 🟡，补充练习环节 |
| 排版不统一 | 图片有编号和没编号混用 | 标注 🟢，统一格式 |
| 小白看不懂 | 专业术语无解释直接使用 | 标注 🟡，添加术语解释 |

## 六、审核人须知

1. **客观公正**：基于标准打分，不带个人偏好
2. **具体可操作**：每个问题必须给出修改建议，不要只说"这里不好"
3. **尊重作者**：用建议语气，不用命令语气
4. **抓大放小**：严重问题优先处理，细节问题可汇总后一次性反馈
5. **保持版本记录**：每次审核保留记录，便于追踪改进过程
